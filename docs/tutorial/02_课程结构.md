# 课程结构与时间轴

建议主视频控制在 18–22 分钟。镜头重点放在导演台的输入、任务列表和最终提示词；模型等待过程全部剪掉，只保留同一提示词在本地与闭源/API模型中的结果对照。

| 时间 | 章节 | 教学目标 | 画面 |
|---|---|---|---|
| 00:00–01:00 | 先看结果 | 同一提示词，两种成图执行端 | 成品对比 |
| 01:00–03:00 | 五类输入 | 分清用户提示、世界观、用途、视觉方法、参考图创意 | 导演台输入区 |
| 03:00–05:00 | 两张接法图 | 说明本地写作与 API①/API②的差别 | 两张接法图 |
| 05:00–09:00 | 提示词核心演示 | 从空文本到世界观、用途和创意组合 | 导演台与任务列表 |
| 09:00–12:00 | 参考图创意 | 解释语义迁移，不是洗图或 latent 控图 | 参考图分析链 |
| 12:00–15:00 | 最终提示词 | 检查任务数量、结构、确定性描述和文字要求 | 最终提示词输出 |
| 15:00–19:00 | 本地 vs 闭源/API | 用同一份提示词比较语义理解和成图执行 | 两组结果 |
| 19:00–21:00 | 易错点与总结 | 说明模型差异、冲突组合和故事分镜边界 | 错误截图与仓库 |

## 可拆分的短视频

1. 《用途和视觉方法到底改变了提示词什么》
2. 《参考图不是洗图：创意机制如何迁移》
3. 《同一提示词，本地模型和闭源模型差在哪》
4. 《API①读图、API②写提示词，为什么要分两次》

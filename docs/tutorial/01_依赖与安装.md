# 依赖与安装

这期视频的重点是提示词功能。只要能打开现有导演台并看到节点输出，就可以录制；不要为了视频临时搭建一套新的复杂工作流。

## 最小环境

| 组件 | 用途 | 是否必须 |
|---|---|---|
| [ComfyUI](https://github.com/Comfy-Org/ComfyUI) | 节点运行环境 | 必须 |
| [ZF-ComfyUI-PromptDirector](https://github.com/Z-yaofang/ZF-ComfyUI-PromptDirector) | 用途、视觉方法、世界观和任务生成 | 必须 |
| [ZF-ComfyUI-Helper](https://github.com/Z-yaofang/ZF-ComfyUI-Helper) | 世界观动态单选、空文本、尺寸和辅助接线 | 推荐 |
| OBS Studio | 录屏与收音 | 录制时必须 |

现有工作流还可能使用 KJNodes、rgthree、Easy-Use、StringOps、JDCN、Comfyroll 等节点包。它们属于运行环境依赖，视频只需在画面中说明“按当前工作流的缺失节点提示安装”，不需要逐个讲解。

## 两条写作路线

### 本地路线

需要支持图片输入的本地多模态模型和本地文本模型，例如 llama-cpp_vllm、匹配的 GGUF 与 mmproj。它适合展示离线、可控和低调用成本；不同本地模型的视觉理解与长文本写作能力会有明显差异。

### API 路线

需要 `comfyui-FOK_API_tools` 或同类 API 节点，以及一个支持图片输入的 API 模型。API①负责读图和结构化分析，API②负责把导演台任务写成最终提示词。API 密钥只放在本机密钥文件中，视频中不要打开或录入密钥。

## 录制前检查

1. 打开你平时已经验证过的导演台工作流；
2. 确认 `ZF Prompt Director` 可以输出系统提示词和任务列表；
3. 准备同一份提示词将要使用的本地成图模型和闭源/API模型；
4. API 只提前测试连通性，不在录屏中展示账户、密钥或私人路径；
5. 把两张接法图放在视频素材文件夹中，录制时作为全景和局部对照。

这里不提供新的教学 JSON。观众应把本教程中的接口原则套回自己的现有工作流，避免把“模型文件名”误认为插件功能的一部分。
